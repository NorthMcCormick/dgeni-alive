$routeSegmentProvider.Pointer

  1. - type in module route-segment

Segment traversal element.

Each $routeSegmentProvider#segment call creates new navigation pointer

Methods

  • segment(name, params, resolve, params.untilResolved, params.resolveFailed, params.default);

    Adds new segment at current pointer level.

    Parameters

    Param Type Details
    name String

    Name of a segment.
    params Object

    Segment's parameters hash. The following params are supported:

    Param Type Details
    template Stringfunction()

    provides HTML for the given segment view;
    templateUrl Stringfunction()

    template to fetch from network via this URL;
    controller Stringfunction()

    cotroller attached to the given segment view when compiled and linked, this can be any controller definition AngularJS supports;
    controllerAs String

    controller alias name, if present the controller will be published to scope under the controllerAs name
    dependencies Array.<String>

    array of route param names which are forcing the view to recreate when changed;
    watcher function()

    $watch-function for recreating the view when its returning value is changed;
    resolve Object.<function()>

    hash of functions or injectable names which should be resolved prior to instantiating the template and the controller;
    params.untilResolved Object

    alternate set of params (e.g. template and controller) which should be used before resolving is completed;
    params.resolveFailed Object

    alternate set of params which should be used if resolving failed;
    params.default Boolean

    when set to true this child segment should be loaded by default when no child is specified in the route.

    Returns

    $routeSegmentProvider.Pointer

    The same level pointer.
